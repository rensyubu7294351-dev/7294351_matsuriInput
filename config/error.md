スプレッドシートにアクセスできません: error:1E08010C:DECODER routines::unsupported                          

⏺ GOOGLE_PRIVATE_KEYの形式がVercelで正しく設定されていないことが原因です。

  原因

  Vercelに秘密鍵を貼り付けるとき、前後の " （ダブルクォート）を含めてしまうと壊れます。

  ❌ 間違い: "-----BEGIN PRIVATE KEY-----\nMIIE..."
  ✅ 正しい:  -----BEGIN PRIVATE KEY-----\nMIIE...
なお、envはダブルクォートをつけたままでOKです。

  理由は場所によって扱いが違うためです：

  ┌──────────────────┬────────────────┬─────────────────────────────────┐
  │       場所       │ ダブルクォート │              理由               │
  ├──────────────────┼────────────────┼─────────────────────────────────┤
  │ .envファイル     │ あってもOK     │ Next.jsが自動で取り除いてくれる │
  ├──────────────────┼────────────────┼─────────────────────────────────┤
  │ Vercelの環境変数 │ 不要・含めない │ そのまま値として使われるため    │
  └──────────────────┴────────────────┴─────────────────────────────────┘
  ---
  修正手順

  Vercelで GOOGLE_PRIVATE_KEY を修正

  1. Vercel →「Settings」→「Environment Variables」
  2. GOOGLE_PRIVATE_KEY を編集
  3. 値を一度全部消す
  4. サービスアカウントのJSONファイルを開く
  5. "private_key" の値をコピー（前後の " は含めない）

  JSONファイルの中身はこんな形です：
  {
    "private_key": "-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----\n"
  }

  コピーする範囲はここだけです：
  -----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----\n

  6. Vercelに貼り付けて「Save」
  7. 「Deployments」→「Redeploy」

  ---
  修正後に管理画面を開いて「シートを初期化する」ボタンが表示されるか確認してください。
