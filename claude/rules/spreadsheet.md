# スプレッドシート仕様

## 概要
- Googleフォームの回答が自動でスプレッドシートに記録される
- お祭りごとに新規作成される（シートIDは config/ フォルダで管理）
- 列構成・中身の変更禁止（後日別作業で使用するため）

## 列定義（左から順番通りに記載）

| 列 | 項目名 | 内容 | 備考 |
|----|--------|------|------|
| A | タイムスタンプ | フォーム回答日時 | 自動入力 |
| B | メールアドレス | 回答者のメールアドレス | 自動入力 |
| C | あだ名 | メンバーのあだ名 | LINE表示名と異なる場合あり・要注意 |
| D | 氏名 | メンバーの本名 | |
| E | ファミリーの所属 | 所属ファミリー名 | プルダウンで選択・13パターン（下記参照） |
| F | 振り（1つ目） | 4パターンから1つ選択 | スプレッドシート上の列名は「振り」 |
| G | 振り（2つ目） | 3パターンから1つ選択 | スプレッドシート上の列名は「振り」。F列と同名のため注意 |
| H | 参加・不参加 | 出欠の回答 | この列で参加・保留を振り分ける |
| I | 絶対練の参加有無 | 絶対練への参加可否 | |
| J | 不参加の理由 | 不参加の場合の理由 | 任意記述 |
| K | 再送チェック | フォームを再送する際にメンバーが手動でチェックを入れる | スプレッドシート上には「再送です」と表示される |
| L | 備考欄 | 自由記述 | |

- 右振り
- 左振り

### 参加・不参加（H列）
- 参加
- 保留（参加より）
- 保留（不参加より）

## 振り分けルール
- 「参加」→「参加LINEグループ」に招待
- 「保留（参加より）」→「保留LINEグループ」に招待
- 「保留（不参加より）」→「保留LINEグループ」に招待

## 注意事項
- あだ名（C列）とLINE表示名が一致しないケースが頻発する
  名寄せロジックを実装する際は目視確認の補助機能も併せて設計すること
- 再送チェック（K列）はメンバーがフォーム再送時に自分でチェックする項目のため
  アプリ側から自動で書き込み・上書き禁止
- F列とG列はスプレッドシート上どちらも列名が「振り」で同一
  コードで列を参照する際はF列・G列と列番号で指定すること
  列名で参照すると誤作動の原因になるため禁止
